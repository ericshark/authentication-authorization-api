# Recap of lessons and work completed for 4.13.26

- Clarified my use of Alembic. Learned that in this specific case, the SQL has a specific data type that needs to be created in SQL, not as easy as altering the column. There needs to be manual input into the actual versions file. When you type into models.py, create an auto-generated file and then try to upgrade and get an error. Best case to fix it is to simply delete that file, make your changes to models.py, and regenerate that file; then you could run upgrade.

- FastAPI caches dependencies within a single request by default (use_cache=True). If get_current_user depends on get_db and your route also depends on get_db, FastAPI runs get_db once and passes the same session to both — only one connection is opened. Two connections only happen if you use two different db functions, manually instantiate a session, or explicitly set use_cache=False. The real reason to verify JWT inline instead of using get_current_user is to avoid an unnecessary database query — if your route only needs the user_id from the token, fetching the full user object from the DB is a wasted round trip, not a wasted connection.

-To define roles safely without typos and allow enumeration, use Python's enum class as a parent. For SQLAlchemy, import Enum from SQLAlchemy and wrap your enum class in it inside mapped_column — this tells SQLAlchemy and Postgres to treat the column as a proper enum type and handle serialization. In the Mapped type hint you still use the Python enum type directly.
