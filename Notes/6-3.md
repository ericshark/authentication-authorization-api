# For Email:

route called
→ generate token with secrets.token_hex(32)
→ store token in Redis with TTL (expiry)
→ call task.delay(email, username, token) ← returns immediately
→ return response to user

Celery worker (separate process) picks up task:
→ render HTML template with Jinja2 (plug variables into template)
→ send rendered HTML via Resend
→ user receives email with link containing the token

user clicks link → hits verification route
→ extract token from link
→ look up token in Redis
→ if found → verify user, delete token from Redis
→ if not found → expired or invalid, reject

# For oauth2:

oauth instance created with client_id, client_secret,
redirect_uri (where Google sends user back),
scope (what info you want — email, profile, openid)

Step 1 — user hits /auth/google route:
Authlib builds authorization URL pointing to Google
generates random state value, stores in session
redirects user's browser to Google's login page

Step 2 — user logs in on Google:
entirely on Google's servers
your app is uninvolved
user approves permissions

Step 3 — Google redirects back to your callback:
browser hits /auth/google/callback
URL contains a short-lived one-time CODE (not a token)
also contains state for CSRF verification

Step 4 — your backend exchanges code for tokens:
Authlib verifies state matches what was stored
makes server-to-server POST to Google (user never sees this)
sends: code + client_id + client_secret
Google responds with:
access_token → permission to call Google APIs (you don't need this)
id_token → JWT signed by Google proving identity
userinfo → parsed user data: email, name, google_id (sub)

Step 5 — create or find user:
check SocialAccount table for provider="google" + provider_user_id
found → get linked user, log them in
check User table by email
found → link Google to existing account, log them in
neither found → create new User (password=None) + new SocialAccount

Step 6 — issue your normal JWT or session cookie
OAuth is done — your regular auth system takes over

# http cookies settings

HttpOnly → JavaScript on the page cannot read the cookie
closes XSS token theft
does NOT help with CSRF

Secure → cookie only sent over HTTPS
closes network interception
does NOT help with CSRF

SameSite → cookie only sent when request comes from same domain
closes CSRF
does NOT help with XSS
